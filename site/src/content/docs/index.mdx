## One workload, from first commit to production

<p class="hero-note">Follow a single workload through its life. The same agent organization is present the whole way - reviewing while you build, reviewing itself at the handoff, and running the control loop once it is live. You stay in the loop through the channel you already use, and you are asked for approvals, not toil.</p>

## What autonomy earns you

<ul class="impact-list">
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>~70-80% of events resolved deterministically.</strong> Rules,
    policies, and checklists decide the repeatable majority. No model call.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>~15-20% resolved by lightweight pattern reuse (T1).</strong>
    Embedding similarity to past resolved incidents, small classifiers, no
    frontier model.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>Only ~5-10% reach the reasoning tier (T2).</strong> Mixed-model
    cross-check plus deterministic verifier - the model proposes, the
    verifier disposes.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>Every autonomous action carries four invariants.</strong>
    Stop-condition · rollback path · impact-scope limit · audit-log entry.
    Missing any of the four ⇒ the action requires human approval, not automatic execution.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <strong>New capabilities start in observation mode, never as a surprise.</strong>
    Enabling changes is a separate, measurable gate against the Phase 0
    baseline.
  </li>
  <li>
    <span class="impact-tick" aria-hidden="true">↳</span>
    <em>All figures above are design targets, not measured results.</em>
    Forward Deployed Agents never claim a multiplier without a paired baseline
    measurement.
  </li>
</ul>

## One core, three domains

<p class="hero-note">Each domain shares the same event-driven, risk-gated core; they differ only in the rules and actions they load.</p>

<CardGrid stagger>
  <Card title="Change Safety · Phase 1" icon="pencil">
    Rule catalog, T0 policy gate, fix PRs. The change check starts in observation mode, then enables changes.
  </Card>
  <Card title="Resilience · Phase 3" icon="rocket">
    Scheduled resilience testing, DB DR drills, impact-scope-bounded chaos experiments - always with a stop-condition and rollback path.
  </Card>
  <Card title="Cost Governance · Phase 3" icon="approve-check">
    Cost anomaly detection, right-sizing PRs, budget guardrails per resource group. Automatic execution or human approval follows the risk classification.
  </Card>
</CardGrid>

## Meet the agent organization

<p class="hero-note">Fifteen named agents run the control plane as a single organization - each owns a role and hands off through a schema-checked event bus. It is an org chart, not a metaphor: the judge never executes, and every action is initiated, judged, approved, executed, and audited by distinct agents.</p>

<CardGrid>
  <Card title="Odin - command" icon="star">
    The master planner. Cross-vertical arbitration and the final tie-break before a decision is finalised.
  </Card>
  <Card title="Judgment and execution" icon="approve-check">
    **Forseti** judges (automatic execution / human approval / denial), **Thor** dispatches and executes, **Var** carries the human approval, **Vidar** owns rollback and DR failover. No agent both judges and executes.
  </Card>
  <Card title="Sensing" icon="magnifier">
    **Huginn** collects and correlates events; **Heimdall** watches health and drift. Deterministic-first - never an LLM in the hot path.
  </Card>
  <Card title="Conversation and audit" icon="comment">
    **Bragi** translates operator questions into tool calls; **Saga** writes the append-only audit trail and escalates handoffs.
  </Card>
  <Card title="Governance" icon="setting">
    **Mimir** owns the rule catalog, **Norns** learns from outcomes, **Muninn** holds long-term memory.
  </Card>
  <Card title="Domain specialists" icon="puzzle">
    **Njord** (cost), **Freyr** (capacity), and **Loki** (chaos) advise the judge with domain analysis - they never execute directly.
  </Card>
</CardGrid>
