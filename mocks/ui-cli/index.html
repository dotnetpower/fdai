FDAI - operator-console - staging
reading status... read-only by default - streaming is just how it reports
Static design mock (no build, no backend). The streaming text, the activity chart, and the bars are just how it reports - the real console reads the same state from read-only console-tool calls, and the assistant only narrates real numbers, never makes them up. Approvals happen by opening a pull request (approve = open a PR, decline = nothing changes, just logged); you can't approve your own request. Plain wording is primary; the precise internal term (for example T0) is shown dimmed beside it. English-only, customer-agnostic, all values synthetic. See .github/instructions/app-shape.instructions.md - Operator console.
