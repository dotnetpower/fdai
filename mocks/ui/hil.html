Rules of this queue

Hard invariants - the console shows state; the risk-gate and the executor enforce them.

Approval ≠ execution

Approver and executor are distinct principals; no self-approval. The executor holds the only privileged identity.

Break-glass is audited

HIGH+ items may bypass the second approver via break-glass, but every use is a post-hoc-reviewed audit entry.

Timeout ≠ approval

A TTL expiry is a no-op with an audit entry, not an implicit yes.

Every item has all four

Stop-condition, rollback path, blast-radius cap, and audit entry - missing one blocks the enforce promotion.
